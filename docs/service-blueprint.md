# HappyTails — карта сервисов и целевая структура (рынок Узбекистана)

> Сопоставление полного набора сервисов HappyTails с тем, что уже реализовано в репозитории, и единая логичная структура трёх приложений: **Couples** (клиент, Telegram mini-app), **Vendor** (кабинет подрядчика, web), **Admin** (внутренняя панель, web).
> Читать вместе с `docs/service-spec.md`, `docs/prompts/improvement.md` и `DESIGN.md`.

---

## 1. Принцип связности

Каждый **сервис** живёт как сквозная тройка:

```
Couples (потребление) ── Vendor (предложение/поставка) ── Admin (каталог · модерация · финансы)
```

- **Сервисы со специалистом** (ветеринар, кинолог, страховка): клиент заказывает → подрядчик оказывает → admin верифицирует и считает финансы.
- **Контентные сервисы** (статьи, курсы, видеоуроки, новости, породы): автор/эксперт создаёт во Vendor → admin модерирует → клиент читает в Couples.
- **Каталожные сервисы** (корма): бренд/магазин ведёт каталог во Vendor → admin модерирует → клиент подбирает/сравнивает в Couples.

Это правило задаёт, какие экраны нужны в каждом приложении для каждой фичи.

Легенда статуса: ✅ есть · 🟡 частично/мок · 🔴 нет.

- **Идентичность и роли:** `users` (telegram_id, phone, locale, roles[]), привязка `owner_id ↔ user_id`.
- **Специалисты:** расширить `vets` полем `specialist_type (vet|cynologist)` или ввести `specialists`.
- **Предложение:** `vendor_services`, `vendor_slots`, `vendor_payouts`.
- **Заказы/деньги:** `payments`, доработать `orders` (полный жизненный цикл), `reviews`, `disputes`.
- **Контент-движок:** единый `content_items (type: article|guide|checklist|course|video_lesson|news|interview, locale{uz,ru}, status)` вместо разрозненных; `video_lessons` (если отдельно).

- **Страховка:** `insurance_plans`, `insurance_policies`, `insurance_claims`.
- **Диагностика:** `symptom_rules` (база знаний).
- **Прочее:** `subscriptions`, `notifications`.

Соответствующие роуты — по образцу `docs/prompts/{couples,vendor,admin}.md` (контракты существующих эндпоинтов не менять).

## 8. Порядок реализации (этапы)

Безопасность и фундамент — **раньше новых фич** (см. `docs/prompts/improvement.md`).

**Этап 0 — Фундамент** *(improvement.md P0–P1)*
Реальная auth (`users` + Telegram initData), роли подрядчиков, единый контент-движок, жизненный цикл заказа + один реальный платёжный провайдер, консолидация `packages/shared`.

**Этап 1 — Здоровье (ядро ценности)**
Кинолог онлайн (обобщение консультаций), Подготовка к визиту и Курс экстренной помощи (как контент), достроить Страховку (планы/полисы + партнёр + Admin).

**Этап 2 — Питание**
Сравнение и Проверка корма, каталог во Vendor (бренд/магазин) + модерация в Admin.

**Этап 3 — Обучение и уход**
Видеоуроки дрессировки, Справочник пород, Новости и интервью, Школа владельца как курс; категория «Уход».

**Этап 4 — Сообщество**
Доработать Места и Добрые дела (бэкенд + Admin), затем AR «Приют доброты».

**Этап 5 — Кабинет и удержание**
Подписка/рассылка, Контакты и поддержка, Диагностика по симптомам (с осторожностью — мед. ответственность), аналитика.
