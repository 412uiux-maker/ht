# Промт-проект: couples — сторона спроса (HappyTails)

> («Реализуй по docs/prompts/couples.md, начни с Milestone 1»). Сначала прочитай `CLAUDE.md`,
> `PRODUCT.md`, `DESIGN.md`, `docs/service-spec.md` (§3 данные, §4 couples, §8 процессы).

## Роль и цель

Ты строишь **клиентское приложение couples** — Telegram mini-app для владельцев питомцев в Узбекистане. Цель: дать владельцу быстро решить задачу (консультация, подбор корма, обучение) с понятной оплатой и ощущением доверия. Якорный сервис — **онлайн-консультация ветеринара**.

## Контекст

- Аудитория, тон, принципы — `PRODUCT.md`. Главная эмоция — доверие; контекст — мобильный, часто стрессовый.
- Визуал, токены, компоненты, бренд — `DESIGN.md`. Тёплый коралл/оранж, Onest, крупные скругления, pill-кнопки, мягкие тени.
- Локали uz/ru, оплата Click/Payme/Uzum, статусы заказа — канон из `service-spec.md`.

## Технологии

- React + **Telegram Web Apps SDK** (mini-app), запуск из бота (`apps/telegram-bot`).
- Авторизация через Telegram `initData` (валидация подписи на стороне `services/api`).
- Данные — через API `services/api` (на старте допустимы моковые данные за реальным контрактом).
- Токены/типы/i18n — из `packages/shared`. GSAP для микроанимаций (easing power2/power3, уважать reduced-motion).

## Объём MVP (волна 1)

Профиль питомца · каталог услуг · карточка услуги/подрядчика · оформление + оплата · чат онлайн-консультации · подбор корма · обучение (гайды/чек-листы) · добрые дела · места с питомцем. Полный список — `service-spec.md` §4.3, §10.

## Экраны (mini-app)

1. Онбординг — язык (uz/ru), вход через Telegram, создание профиля питомца.
2. Главная — персональная лента: рекомендованные услуги, контент, баннер «добрых дел».
3. Каталог услуг — фильтры: категория, город, цена, рейтинг.
4. Карточка услуги/подрядчика — описание, цена (tabular-nums), отзывы, бейдж верификации, CTA.
5. Оформление заказа — выбор питомца, времени, оплата.
6. Чат консультации — текст, фото, файлы, статус, итоговое заключение.
7. Мои заказы — активные/история, статусы, повтор.
8. Подбор корма — анкета → результат → переход к покупке.
9. Обучение — курсы/гайды/чек-листы, прогресс.
10. Добрые дела — инициативы, участие/донат.
11. Места с питомцем — карта/список.
12. Профиль — питомцы, язык, способы оплаты, история.

## Данные (что нужно couples; полная модель — service-spec.md §3)

`User`, `Pet`, `Service`, `Vendor` (для отображения), `Order`, `Consultation`, `Payment`, `Review`, `ContentItem`.

## API-контракты (минимум для среза)

- `POST /api/auth/telegram` — валидация initData → сессия/JWT.
- `GET /api/pets` · `POST /api/pets` — питомцы пользователя.
- `GET /api/services?category=&city=&sort=` · `GET /api/services/:id` — каталог/карточка.
- `POST /api/orders` (service_id, pet_id, scheduled_at) → `Order(created)`.
- `POST /api/payments` (order_id, provider) → checkout/redirect; webhook двигает статус.
- `GET /api/orders` · `GET /api/orders/:id` — список/детали со статусами.
- `GET /api/consultations/:id/messages` · `POST .../messages` — чат.
- `POST /api/food/recommend` (анкета) → рекомендация.
- `GET /api/content?type=` — обучение/гайды.

## Дизайн и качество

- Использовать CSS-переменные из `DESIGN.md`; шрифт Onest; компоненты (pill-кнопки, service-карточки, статус-чипы, чат-пузыри).
- Все строки — uz/ru через i18n, запас +30% длины.
- a11y: контраст AA, видимый фокус, цель ≥44px, `prefers-reduced-motion`.
- Обязательны состояния loading / empty / error в каждом потоке (спокойные, с подсказкой следующего шага).

## Definition of Done (для среза)

Поток проходит end-to-end на uz и ru; оплата работает (sandbox); статусы заказа меняются корректно; экраны адаптивны под мобильный; контраст проверен; нет вложенных карточек; нет хардкод-строк.

## Milestone 1 — первый сквозной срез «Онлайн-ветеринар»

1. Каркас mini-app + авторизация через Telegram initData.
2. Профиль питомца (вид, возраст, кличка) — создание/выбор.
3. Каталог услуг (категория «онлайн-ветеринар») + карточка с ценой, рейтингом, бейджем.
4. Оформление заказа → `Order(created)` → оплата (sandbox Click/Payme) → `paid`.
5. Чат консультации с заглушкой подрядчика + итоговое заключение; статусы до `completed`.
6. Экран «Мои заказы» со статусами. Состояния loading/empty/error. uz/ru.

## Не-цели (волна 2)

Сравнение/проверка кормов, кинолог/видеоуроки, страхование, диагностика по симптомам, AR-игра. Не реализовывать сейчас.
